## プロジェクト概要

Gentnerの構造写像理論（Structure-Mapping Theory, 1983）をLLMパイプラインとして実装し、課題入力から構造的アナロジーに基づくソリューション提案を自動生成するシステム。

## 思想的基盤（最重要 — 全ての設計判断に適用）

1. **アナロジーは発見のツールであり、正当化のツールにしない** — アナロジーから得たアイデアは「仮説」であり、出自に固執しない。Step 5でアナロジーの出自を剥がしてフラットに評価する設計はこの原則の直接的実装。

2. **エレガントな事前設計より泥臭い反復** — 完璧なアーキテクチャを設計してから実装するのではなく、動くものを先に作ってパターンを見つけてから構造化する。「make it work, make it right, make it fast」の順。

3. **型は帰納的に作る** — 実戦から抽出されたパターンを事後的に構造化する。演繹的に設計された「美しい型」は現実に負ける。

4. **アナロジーの有効射程は事前に判定できない** — 完全な監査は計算論的に不可能。「バカだけど安い検査を大量に走らせる」方針でノイズを弾く。

## パイプライン構造

```
[課題入力] → [Step1: 構造抽出] → [Step2: 類似構造探索(近縁5+遠方5)]
  → [Step3: Semantic SME(ランキング)] → [Step4: 候補推論生成]
  → [アイデアバンク(出自剥離)] → [Step5: クロス立案] → [最終提案]
```

## Gentner構造写像理論の実装上の要点

- **マッピング対象は「関係」であり「属性」ではない** — オブジェクトの性質ではなく、オブジェクト間の関係をマッピングする
- **体系性原理** — 孤立した関係ではなく、高次の関係（因果、依存等）で結ばれた関係のシステムが優先される
- **述語はドメイン固有のまま保持** — 抽象的な動詞（cause, generate等）に変換しない。「貪食する」「証券化する」等の生々しい述語を保持しないと、Semantic SMEでスコア差が出ない

## 既知の課題

- **表現のギャップ（Representation Gap）**: LLMの埋め込み空間の情報をテキストに圧縮すると次元の豊かさが失われる。Phase 2でグラフ埋め込み（GNN）での対処を検討
- **Greedy Matching**: 現SMEは1対多マッチング。排他的マッチング（ハンガリアン法）の導入はPhase 2
- **アラインメントの影響**: Step 2（類似構造探索）でフロンティアモデルが「人間にとって自然な組み合わせ」に偏る可能性。A/Bテストで検証
